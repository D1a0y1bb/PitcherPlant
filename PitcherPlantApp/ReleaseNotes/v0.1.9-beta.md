### Build

PitcherPlant v0.1.9-beta removes the status-bar quick panel and tightens validation around localization resources after the settings cleanup.

- Version: 0.1.9-beta
- Build: 25
- Release tag: v0.1.9-beta
- Distribution target: ad-hoc signed GitHub beta artifacts when Developer ID secrets are not available

### User-Facing Changes

#### Status Bar

- The macOS status-bar leaf icon and its quick panel have been removed.
- Recent audits, recent reports, refresh, new audit, start or cancel audit, settings, open app, and quit are no longer exposed through a status-bar extra.
- The normal app window remains the primary entry point for workspace, audit, report, settings, and update workflows.

#### Settings

- The Settings page no longer shows the status-bar panel toggle.
- Stored app settings no longer include the old status-bar visibility field.

#### Localization

- Runtime localization keys and the `.xcstrings` catalog were cleaned together so removed status-bar text does not linger.
- `Localizable.xcstrings` is validated as a string catalog JSON resource and through Xcode's `xcstringstool` compile path. It is not treated as a plist for `plutil -lint`.

### Validation

Local validation before tagging covered:

- `xcodegen generate --spec PitcherPlantApp/project.yml`
- `git diff --check`
- residual reference search for the removed status-bar scene, view, setting field, and localization keys
- `jq empty PitcherPlantApp/Resources/Localizable.xcstrings`
- `swift test --package-path PitcherPlantApp`
- `xcodebuild -project PitcherPlantApp/PitcherPlantApp.xcodeproj -scheme PitcherPlantApp -configuration Debug -destination 'platform=macOS' build`
- `xcodebuild -project PitcherPlantApp/PitcherPlantApp.xcodeproj -scheme PitcherPlantApp -destination 'platform=macOS' test`
- `xcodebuild -project PitcherPlantApp/PitcherPlantApp.xcodeproj -scheme PitcherPlantApp -destination 'platform=macOS' -configuration Release build`
- `./PitcherPlantApp/script/build_and_run.sh --verify`

### Distribution Notes

- Public beta artifacts may be ad-hoc signed and not notarized when Developer ID secrets are unavailable.
- The DMG contains `PitcherPlant.app` and an Applications drag-and-drop shortcut.
- Expected artifacts include ZIP, DMG, Sparkle appcast, xcarchive, dSYM archive, and SHA-256 checksums.
- Gatekeeper may require Control-click > Open, System Settings > Privacy & Security > Open Anyway, or removing quarantine for local testing.
