### Release Overview

PitcherPlant v0.2.1-beta is a beta-channel maintenance release for the native macOS audit workbench. It focuses on audit assistant configuration safety, Settings polish, launch-time workspace behavior, and consistent scroll-edge behavior across primary content pages.

- Version: 0.2.1-beta
- Release tag: v0.2.1-beta
- Channel: beta / RC channel through `appcast-beta`
- Update feed: `https://github.com/D1a0y1bb/PitcherPlant/releases/download/appcast-beta/appcast.xml`
- GitHub Release behavior: published as a pre-release and not promoted to GitHub Latest
- Distribution target: ad-hoc signed GitHub beta artifacts when Developer ID secrets are not available
- Notarization: not notarized in the current ad-hoc beta distribution path

### User-Facing Changes

#### Audit Assistant

- Rebuilds the audit assistant settings surface around explicit provider selection, protocol selection, model selection, API key handling, and data-sharing level controls.
- Adds provider-aware defaults for OpenAI, Anthropic, Gemini, MiniMax, DeepSeek, Kimi, and custom OpenAI-compatible gateways.
- Keeps custom gateway setup explicit: compatible gateways require a user-provided base URL instead of inheriting a vendor endpoint silently.
- Adds a connection test flow that reports provider, model, HTTP status, and parsed response details in the Settings view.
- Preserves assistant suggestions by report ID and evidence ID so selection changes do not attach a response to the wrong evidence row.

#### Settings and Localization

- Expands Chinese and English localization coverage for assistant settings, provider errors, Settings rows, report issue copy, and runtime validation messages.
- Normalizes assistant credential IDs so switching providers stores API keys under predictable keychain accounts.
- Keeps API keys out of persisted app settings while still supporting provider-specific credential storage.
- Improves Settings row layout, field alignment, control grouping, and disabled states for assistant-related controls.

#### Reports and Evidence Review

- Aligns report center, report content, and library-style pages with the shared `AppPageShell` container so titlebar scroll-edge blur behaves like the already-correct native pages.
- Keeps report assistant actions tied to the active evidence row and selected report.
- Preserves review severity, notes, favorites, and watched flags when evidence flags are toggled or batch review actions are applied.

#### Workspace Launch Behavior

- Stops app startup from creating default audit input or output directories automatically.
- Keeps workspace discovery lightweight on launch and leaves audit folders to explicit user actions.

### Audit and Data Safety

#### External Assistant Requests

- Supports provider-specific request builders for OpenAI Chat Completions, OpenAI Responses, Anthropic Messages, Gemini Generate Content, and MiniMax Chat Completion.
- Rejects unsupported provider/protocol combinations before sending a request.
- Normalizes provider response parsing and token usage metadata where supported.
- Applies data-sharing levels so assistant context can be limited to summary-only, evidence detail, or fuller context.

#### Persistence

- Stores assistant suggestions separately from evidence reviews.
- Persists review state without mixing AI suggestions, manual decisions, and user notes.
- Keeps legacy assistant configuration decoding compatible with current disabled/API-key modes.

### Distribution Notes

- This is a beta release. It is intended for validating the native macOS audit workflow before a stable release.
- Public artifacts may be ad-hoc signed and not notarized when Developer ID signing secrets are not used.
- The expected release assets are ZIP, DMG, Sparkle appcast, xcarchive ZIP, dSYM archive, and SHA-256 checksums.
- The DMG contains `PitcherPlant.app` and an Applications drag-and-drop shortcut.
- macOS Gatekeeper may require Control-click > Open, System Settings > Privacy & Security > Open Anyway, or removing quarantine for local testing.
- Sparkle automatic update installation requires clients that already contain the Sparkle EdDSA public key. Older builds without that key may need a manual install before automatic beta updates can be tested.

### Known Notes

- The app remains macOS 26+ only.
- The current beta distribution path does not represent Apple Developer ID notarized distribution.
- A Swift compiler warning for the deprecated `NSToolbar.showsBaselineSeparator` API may still appear during builds; it does not block this beta release.
