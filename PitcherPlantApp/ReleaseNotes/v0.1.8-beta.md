### Build

PitcherPlant v0.1.8-beta is a large-audit reliability release focused on review-state correctness, database-backed browsing, repeat-audit hygiene, safer exports, and release-channel discipline.

- Version: 0.1.8-beta
- Build: 24
- Release tag: v0.1.8-beta
- Distribution target: ad-hoc signed GitHub beta artifacts when Developer ID secrets are not available

### User-Facing Improvements

#### Evidence Review

- Evidence table selection now uses the same composite review target as the Inspector, context menu, and batch-review actions. Selecting an evidence row keeps the row highlight, detail pane, right-click review command, and batch state aligned.
- Reviewing evidence from the table or context menu keeps existing severity and notes unless the user explicitly changes them, so quick review actions no longer wipe useful reviewer context.
- Report review saves refresh the affected row and Inspector state immediately instead of leaving the user looking at stale review status.

#### Large Reports and Evidence Browsing

- Report libraries, evidence rows, and fingerprint results now use database-backed pagination and search paths instead of repeatedly decoding full report payloads for every list refresh.
- Evidence pages load more rows on demand and preserve responsive filtering while moving through large reports.
- Report search avoids repeatedly loading selected report payloads during text entry, reducing UI stalls when the report library is large.

#### Audit Reliability

- Repeated audits of the same directory now use stable fingerprint identity, reducing self-comparison noise from older scans of the same content.
- Document feature caching is isolated by batch-aware keys so later audits do not overwrite feature rows that still belong to earlier batches.
- Large metadata-collision groups are bounded before pair generation, preventing one common author or field from creating explosive evidence-pair counts.
- Candidate recall limits remain visible in audit summaries when oversized buckets are skipped, making possible recall loss explicit.
- Analyzer and evidence-building loops now check cancellation more frequently, so cancelling a large audit stops work faster.

#### Export Safety

- CSV exports now escape formula-like cell values, including values that begin with whitespace or control characters before `=`, `+`, `-`, or `@`. This reduces the risk of spreadsheet formula execution when users open exported evidence in Excel or Numbers.

#### Settings, Accessibility, and Localization

- The main-window Settings entry no longer conflicts with the system `Command-,` Settings shortcut.
- Icon-only controls and Settings rows now expose clearer accessibility labels, values, and help text.
- Runtime localization keys and the `.xcstrings` catalog are kept in sync by tests so English and Chinese interfaces do not drift silently.

#### Release and Update Safety

- Release packaging requires release notes for real tags before publishing.
- Sparkle appcast signing checks require the EdDSA private key in all release paths that generate appcasts.
- Beta and RC builds stay pinned to the fixed `appcast-beta` feed, while stable builds continue to use the stable GitHub Latest appcast.
- The workflow publishes beta and RC releases as prereleases with `--latest=false` and refreshes the shared beta appcast channel.

- `xcodegen generate --spec PitcherPlantApp/project.yml`

- `./PitcherPlantApp/script/build_and_run.sh --verify`
