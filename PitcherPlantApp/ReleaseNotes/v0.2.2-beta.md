### Release Overview

PitcherPlant v0.2.2-beta is a beta-channel hotfix for audit assistant API key cleanup. It fixes a Keychain migration issue where a legacy assistant credential could survive deletion and later make the Settings view show an API key as saved again.

- Version: 0.2.2-beta
- Release tag: v0.2.2-beta
- Channel: beta / RC channel through `appcast-beta`
- Update feed: `https://github.com/D1a0y1bb/PitcherPlant/releases/download/appcast-beta/appcast.xml`
- GitHub Release behavior: published as a pre-release and not promoted to GitHub Latest
- Distribution target: ad-hoc signed GitHub beta artifacts when Developer ID secrets are not available
- Notarization: not notarized in the current ad-hoc beta distribution path

### User-Facing Changes

#### Audit Assistant API Key Cleanup

- Fixes API key deletion in Settings so the current provider-specific Keychain entry and the older legacy default entry are both removed.
- Cleans up the legacy default Keychain entry after migration, instead of leaving a second copy behind.
- Cleans up a stale legacy default entry when a provider-specific key already exists, so older beta installs do not keep carrying the obsolete account.
- Keeps saved API keys out of UserDefaults, the SQLite database, app resources, release notes, and exported release assets.

### Security and Privacy Notes

- This release does not change where API keys are stored: they remain in the user's local macOS Keychain under the app's audit assistant service.
- Published GitHub release artifacts are built by GitHub Actions from repository source and do not include the developer's local Keychain contents.
- Users who previously tested API key deletion should update to this beta, open Settings, delete the audit assistant key again if one is shown, quit, and reopen to confirm it stays deleted.

### Distribution Notes

- This is a beta release intended to repair the audit assistant Keychain cleanup path before the next stable release.
- Public artifacts may be ad-hoc signed and not notarized when Developer ID signing secrets are not used.
- The expected release assets are ZIP, DMG, Sparkle appcast, xcarchive ZIP, dSYM archive, and SHA-256 checksums.
- The DMG contains `PitcherPlant.app` and an Applications drag-and-drop shortcut.
- macOS Gatekeeper may require Control-click > Open, System Settings > Privacy & Security > Open Anyway, or removing quarantine for local testing.
- Sparkle automatic update installation requires clients that already contain the Sparkle EdDSA public key. Older builds without that key may need a manual install before automatic beta updates can be tested.

### Known Notes

- The app remains macOS 26+ only.
- The current beta distribution path does not represent Apple Developer ID notarized distribution.
