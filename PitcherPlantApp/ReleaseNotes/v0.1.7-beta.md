### Release Highlights

PitcherPlant v0.1.7-beta is a reliability and scale update for audit integrity, large library performance, cancellation responsiveness, localization coverage, and beta-channel update safety.

#### Audit Integrity

- Ordinary audits no longer prune document feature caches that belong to other batches, so running a small one-off audit after a batch audit will not silently erase reusable evidence.
- Directory submissions are now snapshotted into app support storage before analysis, making audit evidence reproducible even if the original folder is moved or edited later.
- ZIP-only submissions that contain no auditable documents now fail early instead of producing a misleading empty report.
- Historical fingerprint reuse is bounded before analysis instead of decoding the whole library up front, reducing the cost of large fingerprint stores.

#### Search, Scale, and Cancellation

- Report search, evidence-row search, and fingerprint search now use database-side search indexes and pagination instead of decoding entire payloads in memory.
- Long analyzer loops and longest-common-substring evidence building now check cancellation more frequently, so cancelling a large audit stops work faster.
- Oversized recall buckets are now surfaced in the audit summary when they are skipped, making potential recall loss visible instead of hiding it behind candidate counts.

#### Review UI, Localization, and Accessibility

- Review commands no longer use bare letter shortcuts, preventing accidental approve/false-positive/ignore/whitelist actions while typing in search fields.
- Report and Inspector copy has been moved through the localization layer, removing mixed Chinese strings from the English interface.
- The runtime localization table and `Localizable.xcstrings` catalog are now kept in sync by test coverage.
- Settings and icon-only controls now expose clearer accessibility labels, values, and help text for keyboard and assistive-technology users.
- The embedded Settings button no longer competes with the system `Command-,` Settings shortcut.

#### Release and Update Safety

- Sparkle appcast generation now requires the EdDSA private key for every release path, including dry runs, instead of falling back to local account signing.
- Beta and RC builds are forced onto the fixed `appcast-beta` channel, while stable builds remain on the GitHub Latest appcast URL.
- Release packaging verifies the built app's `SUFeedURL` and silent update URL before publishing artifacts.

### Validation

- Local preflight covered whitespace checks, release-script syntax, localization catalog parsing, static regression checks, and the SwiftPM test suite.
- The GitHub release workflow also regenerates the Xcode project, runs SwiftPM tests, runs Xcode tests, performs a Release build, packages ZIP/DMG/appcast artifacts, and refreshes the beta appcast channel.
